사용자가 "지금 디자인이 하나도 샵백과 동일하게 변경되지 않았는데?" 지적 후 모든 화면 갭 분석을 다시 수행한 결과.

**Why:** 어제(05-05) #1~#15 ShopBack 리디자인을 했다고 메모리에 기록되어 있었지만, 실제 캡처 비교 시 UX 패턴만 일부 차용하고 ShopBack 시각 디자인은 거의 적용되지 않은 상태였음. 사용자가 직접 갭을 지적하여 재분석 수행.

**How to apply:** 재부팅 후 다시 시작할 때, 반드시 이 우선순위(P1→P2→P3) 순으로 진행. 브랜드 키 컬러는 #FF6B35 (DoubleWin 오렌지) 유지. 콘텐츠는 DoubleWin 기획(3단계 회원/번개장터/우리지역) 그대로, 시각 패턴만 ShopBack 매칭.

## 비교 자료
- 샵백 reference: design-mockup/shopback/01-home ~ 07-region (08-cashback 빈 폴더)
- 더블윈 캡처: design-mockup/qa-2026-05-06/ (게스트 + 로그인 후 데이터 화면 17+ 장)
- 06-market(Earn More) / 07-region 샵백은 에러 화면만 있어 참고 가치 낮음

## 핵심 부재 요소 (P1 — 가장 큰 시각 임팩트)
1. **오렌지 그라데이션 헤더** — 홈 진입 hero, 인사/포인트 영역 통합
2. **Promo carousel 컴포넌트** — 홈/카테고리 재사용, 큰 배너 + 인디케이터 dots
3. **mall 카드 표준 컴포넌트** — 큰 정사각 (실제 브랜드 로고 또는 큰 이니셜) + 이전% 취소선 + 신규/UP 배지

## P2 (구조적 변화)
- 홈 세그먼트 탭 (쇼핑/여행/플레이) + 빠른 액션 칩 row
- 몰 상세 hero (풍경 이미지 슬롯 + 페이지 인디케이터) + 카테고리별 캐시백률 테이블 (단일 % → 5+행)
- 일자별 캐시백 그리드 (실시간/5/6/7/8/9)

## P3 (보강)
- 몰 상세 참고 사항 카드 + "더 많은 세일/쿠폰" pill
- Rates and exclusions 모달 (skeleton 포함)
- 상향 캐시백 보라 pill 배지

## DoubleWin 고유 강점 (유지)
- 검색 빈 상태 (추천 몰 horizontal + 인기 검색어 1~7)
- 몰 상세 3단계 적립 타임라인
- 몰 상세 인기 상품 그리드
- 3단계 회원/티어 시스템 (BRONZE까지 잔여 캐시백 표시)
- 하단 CTA는 검정 대신 오렌지 (브랜드 보존)

## 작업 흐름
P1 → 에뮬 검증 → commit + push → P2 → 검증 → commit → P3 (필요 시 데이터 의존 항목은 EC2 배포 후 재검증).
