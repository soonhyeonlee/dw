## 상황
- 사용자가 DoubleWin 프로덕트 기획서를 전달함 (매우 상세, 전체 앱 리디자인)
- 기존 구현된 기본 캐시백 앱에서 대규모 개편 필요
- Figma Make MCP 연결 시도 중 (인증 문제로 재시작 필요)

## 새 기획서 핵심 변경사항

### 1. 홈 화면 전면 개편 (샵백 스타일)
- **쇼핑몰 경유 리스트 블록**: 쿠팡 등 쇼핑몰 아이콘 리스트 → 선택시 상세 페이지 → 이동하기 버튼으로 실제 쇼핑몰 오픈
- **주제별 상품 블록**: 여러 쇼핑몰에서 특정 카테고리(운동화, 계절상품 등) 선별 노출
- **쇼핑몰별 상품 블록**: 특정 쇼핑몰의 선별 상품 노출
- 블록 시스템: 온/오프, 정렬순서, 아이콘 개수 커스터마이징 가능
- 앱 설치 시 해당 쇼핑몰 앱으로, 미설치시 자체 브라우저로 오픈

### 2. 번개장터 (위탁판매 페이지)
- 더블윈이 생산자와 계약 맺고 기획전 형식 위탁판매
- 배너 블록(1개 상품), 2단 구성 블록, 3단 슬라이드 블록
- 다양한 상품 노출 방식 블록 + 커스터마이징

### 3. 회원 체계 3단계 (기존 2종 → 3단계 계층)
- 협회 (최상위, 추후 삭제 가능하도록 on/off)
- 파트너 (소상공인/관장 등, 협회 선택하여 가입)
- 일반회원 (파트너 선택하여 가입, 미선택도 가능)
- 상위 회원 변경 가능 (승인 불필요)
- 각 유형별 캐시 내역이 다름

### 4. 우리 지역 (완전 신규)
- 지역 학원 정보: 별점, 사진, 댓글 (캐치테이블 참고)
- 읽은 횟수(일반/맘카페 구분), 좋아요(하트) 표시
- 맘카페 전용 쿠폰북: 학원 이용권, 할인권 등
- 파트너가 직접 등록, 고유 일련번호/사용기간 있는 쿠폰
- 사용자가 다운로드 → 마이페이지에서 관리

### 5. 카테고리 탭 (샵백 참고)

### 6. 관심목록 개편
- 관심 쇼핑몰 + 관심 상품 탭으로 분리

### 7. 마이페이지 개편
- 환급 시스템 (샵백 참고)
- 도움말: 샵백 봇, 누락 캐시 도움요청
- 이용가이드 (샵백 참고)
- 파트너 전용 공지
- 내정보: 총캐시/사용캐시/남은캐시, 내 쿠폰/사용내역
- 일반유저/파트너 각각 다른 캐시 내역

## 합의된 구현 우선순위
1. **회원 체계 개편** (협회/파트너/일반) - DB 스키마 변경이 선행되어야 함
2. **홈 블록 시스템** - 쇼핑몰 경유 리스트 + 커스터마이징
3. **번개장터** - 위탁판매 상품 페이지
4. **우리 지역** - 학원 정보 + 쿠폰북
5. **관심목록 개편** - 쇼핑몰/상품 분리

## 현재 진행 상태 (2026-04-29 기준)
- ✅ 1. 회원 체계 개편 — User 엔티티 3단계(협회/파트너/일반), 모바일 4단계 가입 플로우
- ✅ 2. 홈 블록 시스템 — HomeBlock/Mall 엔티티, 블록 기반 동적 홈, 쇼핑몰 경유 페이지
- ✅ 3. 번개장터 — MarketProduct/Exhibition/MarketOrder, 배너/2단/3단슬라이드 블록, 상품상세
- ✅ 4. 우리 지역 — Academy/AcademyReview/Coupon/UserCoupon, 학원 목록/리뷰/쿠폰북
- ✅ 5. 관심목록 개편 — MallWishlist 추가, 관심 쇼핑몰 + 관심 상품 분리
- ✅ 6. 모바일 화면 샵백 스타일 통일 — 홈/마켓/우리지역/쇼핑몰상세/상품상세/마이페이지/캐시백 모두 샵백 스타일 적용 완료. 디자인 토큰: COLORS.ink/primarySoft, 다크 캐시카드 패턴, 세그먼트 탭, 빠른메뉴 4grid 통일
- ✅ 7. 우리지역 학원 상세 화면 — `app/region/[id].tsx` 신규 작성 (헤로 + 학원정보/리뷰/쿠폰 탭, 맘카페 분리 통계, 하단 상담 신청 액션바)
- ✅ 8. 마이페이지 파트너 전용 메뉴 — memberType이 partner/association이면 "파트너 전용" 섹션 자동 노출
- **다음 단계**: 어드민 패널 구현, 실제 데이터 연동, mock에서 API 전환 (현재 cashback/withdrawal API는 연결됨, 우리지역/마켓은 mock)

**Why:** 사용자가 기획서 기반으로 앱 전면 개편 요청. Figma Make로 디자인 생성 → 코드 변환 워크플로우 원함.
**How to apply:** 다음 대화에서 프로젝트 구조 파악 후, 우선순위대로 구현 진행. 1) 회원체계 DB 스키마 → 2) 홈 블록 → 3) 번개장터 → 4) 우리지역 → 5) 관심목록
