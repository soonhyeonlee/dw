# DoubleWin(더블윈) 프로덕트 기획서

---

# 1. 프로젝트 개요

## 1.1 서비스 정의

| 항목 | 내용 |
|------|------|
| 서비스명 | DoubleWin (더블윈) |
| 슬로건 | 쇼핑하면 두 번 이긴다 |
| 서비스 유형 | 크로스플랫폼 캐시백 쇼핑 플랫폼 |
| 플랫폼 | iOS 앱, Android 앱, 어드민 웹 |
| 타겟 시장 | 한국 이커머스 소비자 (20-40대) |

## 1.2 핵심 가치 제안

쿠팡, 네이버, 11번가에 흩어져 있는 쇼핑 혜택을 하나의 앱에서 비교하고, 구매 시 캐시백을 적립하여 현금으로 출금할 수 있는 서비스.

기존 서비스와의 차이:

| 기존 서비스의 한계 | 더블윈의 해결 방식 |
|-------------------|-------------------|
| 각 플랫폼별 캐시백이 분산 | 크로스플랫폼 통합 - 하나의 앱에서 모든 플랫폼 비교 |
| 사용자가 직접 최저가를 검색 | 플랫폼 간 가격 비교를 한 화면에서 제공 |
| 캐시백이 포인트/적립금으로 묶임 | 현금 출금 가능 |
| 유료 멤버십 필요 (와우, 네이버+) | 무료 서비스 |
| 크리에이터 대상 어필리에이트 | 일반 소비자가 직접 캐시백 수령 |

## 1.3 비즈니스 모델

```
사용자가 더블윈 앱에서 상품 클릭
    |
    v
어필리에이트 URL을 통해 쇼핑몰 이동 및 구매
    |
    v
쇼핑몰이 더블윈에 어필리에이트 수수료 지급
    |
    +-- 운영 수익 (수수료의 30~50%)
    |
    +-- 사용자 캐시백 (수수료의 50~70%)
```

수익 예시: 사용자가 쿠팡에서 50,000원 상품 구매 (수수료율 5%)

| 항목 | 금액 |
|------|------|
| 쿠팡 → 더블윈 수수료 | 2,500원 |
| 더블윈 → 사용자 캐시백 (60%) | 1,500원 |
| 더블윈 운영 수익 (40%) | 1,000원 |

| 공백 | 설명 |
|------|------|
| 크로스플랫폼 캐시백 어그리게이터 부재 | 쿠팡/네이버/11번가를 한 곳에서 비교하는 독립 서비스가 없음 |
| 소비자용 캐시백 부재 | 어필리에이트 프로그램은 크리에이터 대상이며, 일반 소비자가 구매로 캐시백 받는 모델이 없음 |
| 가격 추적/하락 알림 통합 부재 | 크로스플랫폼 가격 히스토리 + 하락 알림 + 캐시백을 묶은 서비스 없음 |

# 4. 기능 정의

## 4.1 Phase 구분

| Phase | 목표 | 주요 기능 |
|-------|------|----------|
| Phase 1: MVP | 핵심 캐시백 플로우 완성 + 스토어 출시 | 상품 탐색, 캐시백 적립/출금, 인증, 어드민, 크롤러 |
| Phase 2: Growth | 사용자 리텐션 + 바이럴 성장 | 가격 추적/알림, 게이미피케이션, 추천인, 소셜 로그인, 푸시 알림 |
| Phase 3: Expansion | 수익 확대 + 플랫폼 확장 | 브라우저 확장, 추가 쇼핑몰 연동, 소셜 쇼핑, 금융 리포트 |

## 4.2 Phase 1 기능 상세

### F1. 사용자 인증

#### F1.1 회원가입

| 항목 | 사양 |
|------|------|

|------|------|
| 입력 필드 | 이메일, 비밀번호 |

```

        |     |
        |     +-- 성공 -> 메인 화면
        |     +-- 실패 (만료) -> 토큰 삭제 -> 로그인 화면
```

### F3. 캐시백 시스템

#### F3.1 캐시백 플로우

```
1. 사용자가 상품의 "구매하기" 버튼 클릭
   |
2. 서버에 클릭 로그 기록 + 어필리에이트 URL 반환
   |
3. 인앱 브라우저로 어필리에이트 URL 열기 -> 사용자가 쇼핑몰에서 구매
   |
4. 구매 확정 (어드민이 수동 확인, 향후 웹훅 자동화)
   |
5. 캐시백 트랜잭션 생성 (상태: 대기)
   주문금액 x 수수료율 = 수수료
   수수료 x 캐시백율(50-70%) = 사용자 캐시백
   |
6. 캐시백 확정 -> 사용자 잔액에 반영
```

#### F3.2 캐시백 비율 구조

| 플랫폼 | 어필리에이트 수수료율 | 사용자 캐시백율 | 사용자 실질 캐시백 |
|--------|---------------------|---------------|------------------|
| 쿠팡 | 3-7% (카테고리별) | 수수료의 50-70% | 구매가의 1.5-4.9% |
| 네이버 | 1-5% (카테고리별) | 수수료의 50-70% | 구매가의 0.5-3.5% |
| 11번가 | 1-3% (카테고리별) | 수수료의 50-70% | 구매가의 0.5-2.1% |

#### F3.3 캐시백 상태 흐름

```
대기(pending) -> 확정(confirmed) -> 지급(paid)
      |
      +-> 취소(cancelled)
```

| 상태 | 설명 |
|------|------|
| pending | 구매 추적 완료, 구매 확정 대기 중 |
| confirmed | 구매 확정됨, 사용자 잔액에 반영 |
| paid | 출금 완료 |
| cancelled | 구매 취소/반품으로 캐시백 취소 |

#### F3.4 캐시백 화면 구성

| 영역 | 내용 |
|------|------|
| 잔액 카드 | 현재 잔액 (큰 숫자), 출금 버튼 |
| 요약 통계 | 총 적립액, 총 출금액, 대기 중 금액 |
| 탭 전환 | 적립 내역 / 출금 내역 |
| 적립 내역 항목 | 상품명, 플랫폼, 금액, 상태 뱃지, 날짜 |
| 출금 내역 항목 | 금액, 은행, 상태, 날짜 |

### F4. 출금 시스템

#### F4.1 출금 조건

| 항목 | 조건 |
|------|------|
| 최소 출금액 | 5,000원 (어드민 설정에서 변경 가능) |
| 출금 방법 | 계좌이체 (은행명 + 계좌번호 + 예금주) |
| 처리 기간 | 요청 후 1-3 영업일 (어드민 수동 승인) |
| 출금 수수료 | 무료 (월 3회), 이후 건당 500원 |

#### F4.2 출금 플로우

```
사용자가 출금 요청
  |
  +-- 잔액 >= 최소 출금액? -> 아니오 -> "잔액이 부족합니다" 에러
  |
  +-- 계좌 정보 등록됨? -> 아니오 -> 계좌 등록 화면으로 이동
  |
  +-- 금액 입력 및 확인
  |
  +-- 잔액 즉시 차감, 출금 요청 생성 (상태: requested)
        |
        +-- 어드민 승인 -> 계좌이체 -> 상태: completed
        |
        +-- 어드민 거절 -> 잔액 복구 -> 상태: rejected + 거절 사유
```

#### F4.3 출금 상태 흐름

```
요청(requested) -> 처리중(processing) -> 완료(completed)
      |
      +-> 거절(rejected) -> 잔액 자동 복구
```

## 6.1 첫 구매 캐시백 플로우

```
앱 설치
  |
  v
스플래시 -> 로그인 화면
              |
              +-- 이메일 회원가입
              |     이메일 입력 -> 비밀번호 입력 -> 닉네임 입력
              |     -> 가입 완료 -> 자동 로그인
              |
              +-- 소셜 로그인 (Phase 2)
  |
  v
홈 화면
  |
  +-- 카테고리 선택 또는 검색
  |
  v
상품 목록 (플랫폼 필터 적용 가능)
  |
  v
상품 카드 탭
  |
  v
상품 상세
  |
  +-- 캐시백 안내 확인
  |   "이 상품을 구매하면 1,500원 캐시백"
  |
  +-- "캐시백 받고 구매하기" 버튼 탭
  |
  v
인앱 브라우저 (쿠팡/네이버/11번가 어필리에이트 URL)
  |
  +-- 사용자가 쇼핑몰에서 결제 완료
  |
  v
캐시백 적립 (대기 상태)
  |
  +-- 구매 확정 후 (수일 소요)
  |
  v
캐시백 확정 -> 알림 "1,500원 캐시백이 확정되었습니다"
  |
  +-- 잔액 반영
  |
  v
잔액 5,000원 이상 시
  |
  +-- 출금 요청 -> 금액 입력 -> 계좌 확인 -> 요청 완료
  |
  v
어드민 승인 -> 계좌 입금 -> 알림 "출금이 완료되었습니다"
```

# 11. KPI

## 11.1 핵심 지표

이 지표가 증가하면 사용자 활동(구매)이 활발하다는 의미이며, 수수료 수익도 비례하여 증가.

| MAU | 1,000 | 10,000 | 50,000 |
| D1 리텐션 | 40% | 50% | 55% |
| D7 리텐션 | 20% | 30% | 35% |
| D30 리텐션 | 10% | 18% | 25% |

| 이용약관 | 필수 | 앱스토어 심사 필수 항목 |
| 통신판매업 신고 | 확인 필요 | 캐시백 서비스 해당 여부 법률 검토 |
| 전자금융업 등록 | 확인 필요 | 현금 출금 기능 시 해당 가능 |
| 개인정보보호법 준수 | 필수 | 은행 계좌 정보 수집/저장 관련 |
| 어필리에이트 약관 준수 | 필수 | 각 플랫폼(쿠팡/네이버/11번가) 이용약관 검토 |

# 14. 용어 정의

| 용어 | 정의 |
|------|------|
| 캐시백 | 상품 구매 시 구매 금액의 일부를 사용자에게 환원하는 것 |
| 어필리에이트 | 제휴 마케팅. 사용자를 쇼핑몰로 유도하여 구매 발생 시 수수료를 받는 구조 |
| 어필리에이트 URL | 추적 코드가 포함된 상품 링크. 이 링크를 통한 구매만 수수료가 발생 |
| 클릭 로그 | 사용자가 구매 버튼을 눌러 어필리에이트 URL로 이동한 기록 |
| UPSERT | 데이터가 없으면 INSERT, 있으면 UPDATE하는 DB 연산 |
| MAU | Monthly Active Users. 월간 활성 사용자 수 |
| D7 리텐션 | 가입 후 7일째에 앱을 다시 사용한 사용자 비율 |
| CAC | Customer Acquisition Cost. 사용자 1명을 획득하는 데 드는 비용 |
| Rate Limit | 일정 시간 내 API 요청 횟수를 제한하는 것 |
